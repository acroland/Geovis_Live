Check your understanding (Part 1)

7. What does CRS mean, and why does it matter for mapping and area-based distortion?

CRS stands for “coordinate reference system”, which is a system that defines how spatial data translates to real positions on the Earth’s surface. This is crucial for mapping since it allows us to transform locations on the 3D surface of the Earth so they can be depicted on 2D surfaces, such as a piece of paper or a computer screen. Since locations on a 3D surface, such as the Earth, cannot be perfectly translated to 2D, cartographers must make compromises. Thus, there are many different types of CRS with various characteristics to suit an array of mapping needs. Area-based distortion, one such issue, is the misrepresentation of the size of geographic features that occurs when projecting the 3D surface of the Earth on a 2D plane. Area-based distortions are an important consideration when choosing a CRS, since distortions can cause data to be displayed in misleading or inaccurate ways.

8. What does gdf.total_bounds return, and how would you use it to sanity-check your data?

Returns the coordinates indicating the bounds of all geometry features in a dataset. If the data seems to be displaying incorrectly (wrong locations, missing locations, etc.), this function could be used to confirm that the data includes the desired spatial extent. Basically, it could help with troubleshooting.

9. Which line reprojects the data, and what would happen if WORK_EPSG is invalid?

heatIndex_m = heatIndex.to_crs(WORK_EPSG)

If the variable specifying the CRS was invalid, running this line would not reproject the data, and would result in an error stating that the projection is invalid.


Check your understanding (Part 2)

10. Which parameter controls the number of classes (bins)?

Within the plot parameters:

k=_ (in the example, it was k=5)

11. Which parameter changes the color palette? Give two valid cmap examples.

Within the plot parameters:

cmap=_ (e.g., cmap=”Blues”, cmap=”RdYlBu”)

12. Which parameters control polygon boundary appearance (color and line width)?

Within the plot parameters:

linewidth=_ (line width, e.g., linewidth=0.3)

edgecolor=_ (color, e.g., edgecolor=”white”)

13. What does scheme='Quantiles' mean, and how is it different from 'EqualInterval'?

The “scheme” parameter dictates how the method by which data is classified into bins. The quantile method creates the bins (adjusts the range of data in each bin) so that each bin has an equal number of observations, while the equal interval method creates classes with an equal range of values, regardless of how many observations fall into each bin.


Check your understanding (Part 4)

18. Why must the cartogram field be positive?

The cartogram field must be positive because the distortion of the map features is scaled based on the chosen variable. It wouldn’t make sense conceptually to have negative values.

19. What do max_iterations and max_average_error control?

Max_iterations controls the maximum number of times the transformation algorithm is applied. Max_average_error controls the target average error of the output polygons.

20. What is c.average_error measuring (in plain language)?

The average residual error, which is a measure of how well the resized regions match the data.

21. In the fallback method, why do we use sqrt(value/mean) rather than value/mean directly?

To normalize the scaled values and maintain the shapes of the polygons.

Lab 6 questions

Part 1

  1. What is a DEM, and what does each pixel value represent?
  2. A Digital Elevation Model (DEM) is a representation of the bare ground topography of the Earth’s surface using a raster surface. Since DEMs represent bare ground, they exclude features such as trees and buildings. The value of each pixel represents the average elevation of the corresponding area.
  3. Why do we convert nodata values to np.nan before plotting?
  4. We convert nodata pixels to np.nan so Python actually understands that they’re no data values when running calculations. Different DEMs assign different values to nodata pixels, such as -99999, but Python doesn’t know that, and might try to actually use those values when working with the data. This would greatly skew any results and would cause the data to be visualized inaccurately during mapping. By designating the nodata pixels as nan, we ensure that Python basically ignores those pixels when working with the dataset.
  5. What does “resolution” mean for a raster, and how do you read it from the rasterio transform?
  6. In this specific context, the resolution is referring to the spatial resolution of the data. This represents the real-world dimensions represented by each pixel. The spatial resolution for this DEM is 1 x 1 m (since the CRS meters as its unit of measurement), meaning that each pixel represents a land area of 1 sq meter. The code used to read the resolution from the rasterio transform is: print("Resolution:", (transform.a, -transform.e)). The “.a” part reads the pixel width and “.e” reads the pixel height. Together, those two values tell us the spatial resolution.


Part 2

  1. Which parameter changes the color ramp? Give two valid cmap names.
  2. cmap; viridis, gist_earth
  3. What do vmin and vmax do, and why might a percentile stretch be preferable to min/max?
  4. The vmin and vmax parameters specify the lowest and highest values, respectively, displayed using the color ramp. For example, if we had a dataset with a range of 0-100, and set the vmin and vmax to 25 and 75, respectively, pixels with a value of 25 would be set to the start of the color ramp, and pixels with a value of 75 would occur at the end. Everything higher than 75 or lower than 25 will be the same color. We could also set these to percentile values, like we did in the lab. This would be useful in a dataset with outliers, where having a few extreme values might squish the rest of the data toward the center of the color ramp.
  5. If you change the stretch, does it change the underlying data values? Why or why not?
  6. Changing the stretch does not change the underlying data values, only how the data is displayed using the color ramp. This is because stretch is simply a data visualization method. You can change the stretch to change how the same data is shown, meaning a single dataset could look different depending on the stretch parameters. Further, the stretch relies on the underlying data to work, especially when using parameters like vmin and vmax, so it would likely cause a lot of problems if changing the stretch altered the underlying dataset.


Part 3

  1. In one sentence: what does slope measure, and what does aspect measure?
  2. Slope measures the angle of incline, and aspect measures the direction of the slope (the measure in degrees can be translated to cardinal directions)
  3. Why is aspect considered a circular variable?
  4. Aspect is considered a circular variable because its values can range from 0-360 degrees, and restart from 0 once you get all the way through the entire range (basically, once you’ve made a circle, the values restart).
  5. If your DEM is in degrees (lat/lon), what problem can that create for slope/aspect calculations?
  6. The tools expect the horizontal units to be the same as the vertical units; for example, if the elevation (vertical) is measured in meters, the horizontal coordinates are also expected to be measured in meters. This can lead to inaccurate slope/aspect calculations and weird scaling since 1 degree of longitude does not equal 1 degree of latitude.



Part 4

  1. What is the visual purpose of hillshade in cartography?
  2. The visual purpose of hillshade in cartography is to visualize the terrain as it appears on Earth, which is to say, a more realistic appearance. By simulating light and shadow on the DEM surface, we can give the 2D scene a 3D appearance.
  3. Which parameter changes the direction of light? Which changes the sun height?
  4. Direction - azimuth; sun height - altitude
  5. Why might you choose a lower altitude (more shadows) vs a higher altitude (flatter look)?
  6. A lower altitude might be helpful when visualizing a surface with lots of elevation variation, since it would highlight those changes. On the other hand, using a higher altitude might be helpful when visualizing flatter elevations by highlighting only the most dramatic elevation changes.


Part 5

  1. Which line controls the transparency of the hillshade overlay?
  2. ax.imshow(hill, cmap="Greys", alpha=0.6)
  3. If the overlay makes the map too dark, what two things can you adjust?
  4. Alpha (transparency), cmap (you could try changing the color scheme for the underlying DEM)
  5. Why is it helpful to keep a colorbar on the DEM even when adding hillshade?
  6. Sometimes it is not obvious which areas are higher vs. lower, even with the hillshade. For example, what appears to be a river channel running through the center of my composite could be mistaken for peaks of a mountain range instead. This is especially important as an accessibility feature for colorblind individuals, as certain color schemes may not be colorblind-friendly.

        var traceType = point.data && point.data.type;

        if (traceType === 'bar')          return barIds[idx];

        if (traceType === 'scatter')      return idx;   // scatter city_id == array index

        if (traceType === 'scattermapbox' || traceType === 'scattermap') return idx;
